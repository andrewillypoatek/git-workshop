Fork & Clone

Get the instructor's code onto your machine. You'll fork it first (your own copy on GitHub), then clone it locally.

Repo to fork
andrewillypoatek / learning-github
Open on GitHub →
1

Fork the repo

On the repo page, click Fork (top right). Forking creates your own copy on GitHub — a place you control and can push to freely, without touching the instructor's original.

Important — Clone your fork, not the instructor's repo. Your fork URL has your username in it.
2

Copy the SSH URL

On your fork, click the green Code button → select SSH → copy the URL. It looks like:

git@github.com:YOUR-USERNAME/learning-github.git
3

Clone it locally

In your terminal, navigate to where you want the folder, then run:

git clone git@github.com:YOUR-USERNAME/learning-github.git hello-world

The hello-world at the end names the local folder.

4

Enter the folder

cd hello-world

Git commands only work when you're inside a repo folder. This puts you in the right place.

5

Check your status

git status
Expected — "nothing to commit, working tree clean" — means your local copy matches GitHub exactly.

Then open index.html in a browser — double-click it in Finder.
