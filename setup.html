Let's get set up

Before we touch any code, we need Git configured and your machine connected to GitHub. Do this together — instructor leads, everyone follows.

git & identity
1

Verify Git is installed

Open Terminal (Spotlight → "Terminal") and run:

git --version

You should see something like git version 2.x.x.

Nothing? — Mac will likely prompt you to install Xcode Command Line Tools. Click Install and wait — it takes a few minutes, then run the command again.
2

Set your identity

Tell Git your name and email. These appear on every commit you make.

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Use the same email you signed up to GitHub with.

ssh key setup
3

Generate an SSH key

SSH is how your machine proves its identity to GitHub — no password needed every time you push. Run (use your GitHub email):

ssh-keygen -t ed25519 -C "you@example.com"

When prompted:

• File location — press Enter (keep the default)
• Passphrase — press Enter twice (no passphrase)

Expected — "Your identification has been saved in /Users/you/.ssh/id_ed25519"
4

Copy your public key

This copies the key to your clipboard so you can paste it into GitHub:

pbcopy < ~/.ssh/id_ed25519.pub

No output = it worked. The key is now in your clipboard.

5

Add the key to GitHub

Go to github.com/settings/keys and click New SSH key.

• Title: MacBook (or any label that makes sense)
• Key type: Authentication Key (default)
• Key: paste with ⌘V

Click Add SSH key.

6

Test the connection

ssh -T git@github.com
Expected — "Hi YOUR-USERNAME! You've successfully authenticated, but GitHub does not provide shell access."

If you see your username in that message, you're connected. We're ready to go.
