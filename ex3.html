Branch & Pull Request

0:32 → 0:43

Create a separate branch to keep your work isolated from main, then open a Pull Request to propose merging it back.

main
feature/your-name
Your branch lives alongside main — changes are isolated until you PR them in.
1

Create a new branch

git checkout -b feature/your-name

Replace your-name with your actual name, e.g. feature/miguel.

2

Make an edit

Change anything you like — the subtitle in index.html, a color in style.css, whatever you want.

3

Stage and commit

git add . git commit -m "your message here"
4

Push the branch to GitHub

git push -u origin feature/your-name

The -u flag links your local branch to GitHub so future pushes just need git push.

5

Open a Pull Request

Go to your fork on GitHub. You'll see a banner: "Compare & pull request" — click it.

Set the target to main, write a short description, and click Create pull request.

Goal: Everyone opens a PR. Merging is optional today.
Note: You're opening a PR on your own fork. In real teams, you'd PR into a shared repo.
