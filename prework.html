Set up before
the workshop

Complete these steps before we meet — about 10–15 minutes. If anything breaks, don't worry: we'll sort it out at the start of the session.

install & account
1

Install Git

Open Terminal (search for it in Spotlight) and run:

git --version

If you see a version like git version 2.x.x, you're done. If it says "command not found", install it from:

https://git-scm.com/download/mac
Note — On Mac, running git --version may prompt you to install Xcode Command Line Tools. Click Install and wait.
2

Create a GitHub account

Go to github.com and sign up. If you already have one, you're good.

Tip — Use a personal email you'll keep long-term. Your GitHub username is visible publicly.
3

Set up your identity

Tell Git who you are (use your own name and email):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

This is what shows up on your commits.

ssh key setup
4

Generate an SSH key

This creates a secure key pair so you can push to GitHub without a password each time. Run (use your GitHub email):

ssh-keygen -t ed25519 -C "you@example.com"

When asked:

• File location — press Enter for the default
• Passphrase — press Enter twice (no passphrase)

Expected — "Your identification has been saved in /Users/you/.ssh/id_ed25519"
5

Copy your public key

pbcopy < ~/.ssh/id_ed25519.pub

No output means it worked — the key is now in your clipboard.

6

Add the key to GitHub

Go to github.com/settings/keys and click New SSH key.

• Title: MacBook (or whatever makes sense)
• Key: paste (⌘V) your key

Click Add SSH key.

7

Test the connection

ssh -T git@github.com
Expected — "Hi YOUR-USERNAME! You've successfully authenticated, but GitHub does not provide shell access."

If you see that message, you're all set.
