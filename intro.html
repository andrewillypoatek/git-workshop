Before we touch
a single command

Two minutes of context that will make everything else make sense.

// 01 · the difference
git → local tool
  • lives on your machine
  • tracks changes over time
  • works completely offline
github → cloud host
  • stores your repos online
  • enables team collaboration
  • pull requests live here
Analogy: Git is the version control software on your computer. GitHub is the website where you back it up and share it — like Google Drive, but built for code.
// 02 · four words you need
repo
the project folder — everything Git is tracking
commit
a saved snapshot of your files at a point in time
branch
a parallel version — changes stay isolated from main
PR
a proposal to merge a branch back into main
Don't worry if these feel abstract — they'll click once you've done the exercises.
// 03 · the workflow

Everything you do in Git follows this same loop. Today you'll run through it end to end.

fork → clone → branch → edit → commit → push → PR
fork — copy the repo to your GitHub account
clone — download it to your machine
branch — create a safe workspace
edit → commit → push — the core loop, repeated
PR — propose your changes to the team
